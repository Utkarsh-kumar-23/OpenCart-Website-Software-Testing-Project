Project Overview

This project involves testing the OpenCart e-commerce website, ensuring that all functional and non-functional requirements are met. OpenCart is a web-based platform that allows users to browse products, add them to the cart, checkout, and make payments, while administrators manage products, orders, and system settings.

Objective:

Validate system functionality across all modules.
Ensure a secure, responsive, and user-friendly interface.
Identify defects and ensure quality before production release.
Modules Covered
Login / Registration Module
User authentication and session management
Validation for username and password
Logout and security verification
Product Management Module
Add, edit, delete products
Validate mandatory fields and inputs
Product search and stock updates
UI alignment and responsiveness
Cart & Checkout Module
Add to cart and quantity management
Apply coupons and validate billing/shipping
Guest checkout and order summary verification
Mobile and desktop UI validation
Order Management Module
Order placement, cancellation, and partial returns
Invoice generation and email notifications
Order sorting, tracking, and responsive UI
Payment Gateway Module
Multiple payment methods
Validate expired cards, insufficient balance, and retries
Payment confirmation and refund processing
Data encryption and responsive UI
Admin Module
User management, roles, and permissions
Sales reports, system logs, and notifications
Product management, tax, and shipping configuration
Testing Artifacts
Test Plan: Defines strategy, objectives, scope, and approach.
Test Scenarios: High-level testable requirements per module.
Test Cases: Detailed step-by-step instructions including expected and actual results.
Requirement Traceability Matrix (RTM): Ensures complete coverage of requirements.
Bug Reports: Documents defects found during testing with severity, priority, and status.
Testing Approach
Functional Testing: Validate all business workflows and user operations.
GUI / Usability Testing: Verify UI elements are consistent, responsive, and aligned.
Positive & Negative Testing: Ensure correct system behavior for valid and invalid inputs.
Field Validation: Check all mandatory and optional fields for errors and valid data handling.
Security Testing: Validate login authentication, session handling, access control, and data encryption.
Regression Testing: Re-run test cases after defect fixes to prevent new issues.
